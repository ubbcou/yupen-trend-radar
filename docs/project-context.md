# 项目级上下文

定位：本文件是项目级长期记忆，保存稳定事实、产品决策和执行约束。每日鱼盆排名、市场状态和当前方向不写入这里，以 `docs/action-board.md` 为准。

## 项目目标

把猫笔叨公众号文章和文章内的鱼盆表格，转成可持续更新、可回溯验证的趋势买入方向雷达。

项目不追求覆盖所有投资方法，只专精：

```text
趋势买入 = 鱼盆前排 + 站上20日线 + 板块强于大盘
```

文章提供市场解释、催化和风险线索；鱼盆表格提供排名、趋势位置、量能和连续性证据。两者需要相互验证，任何一方都不能单独构成操作结论。

## 产品边界

当前产品输出方向判断，不输出自动交易指令，也不默认扩展到单只股票操作建议。

不做：

- 泛投资资讯聚合
- 实时行情系统
- 主力资金猜测
- 自动交易
- 没有鱼盆表格证据的方向判断
- 与趋势买入模型无关的买入体系

ETF或股票映射可以作为临时研究结果，但在产品规则正式调整前，不进入标准方向雷达主流程。

## 数据口径

每篇文章必须记录三个日期字段：

```text
文章发布日期：文章对外发布的日期
指数鱼盆数据日：指数表实际对应的交易日
板块鱼盆数据日：板块表实际对应的交易日
```

指数表和板块表可能不是同一个数据日，必须逐图读取。方向池依据对应表格数据日生成；文章发布日期只用于归档和解释。

鱼盆表格图片是原始证据。必须从图片核对：

- 指数或板块排名
- 20日线偏离率
- 量比
- 状态转变时间
- 区间涨幅
- 排名变化

缺少图片或图片无法识别时，不生成确定性鱼盆结论。

## 标准更新流

```text
接收新增链接
-> 更新链接记录
-> 抓取并核对正文
-> 若有鱼盆表，抓取并人工读取图片；无表则只归档文章
-> 分别记录指数表和板块表数据日
-> 更新统一观察数据
-> 记录文章对具体方向的支持/反对；未提及保持中性
-> 自动生成指数历史排名
-> 自动生成方向信号快照
-> 自动生成信号生命周期
-> 覆盖当前操作看板
-> 追加历史阅读报告
-> 必要时更新鱼盆指南
-> 校验数据和文档一致性
-> 生成 Web 只读快照
```

## 方向池语义

| 分组 | 判断语义 |
|---|---|
| 主攻 | 满足产品流程中的完整主攻确认条件 |
| 试探 | 核心趋势成立，但至少一项确认条件待满足 |
| 趋势保持 | 趋势仍在，但当前不满足新增条件 |
| 等待 | 只有部分条件改善，尚未形成趋势买入 |
| 回避 | 同时满足产品流程中的后排和水下条件 |

具体阈值和市场基准只在 `docs/product-flow.md` 维护。方向分组描述优先级，动作后缀描述买点；方向成立但触发偏热条件时，必须附加“不追高”。

## 长期维护原则

1. 当前结论覆盖更新，历史证据追加保存。
2. 每篇文章一个总结；进入新阶段后做压缩总结，为后续文章提供上下文。
3. 指数排名按数据日纵向比较，不能只读单日名次。
4. 方法指南必须随着文章和实证进化，不能固定成一次性理论说明。
5. 新信息与旧判断冲突时，记录判断迁移及其证据，不静默改写历史。
6. Web 只消费校验后的生成快照；快照和页面都不能反向覆盖项目事实源。
