# 产品流转流程

目标：把“猫笔叨文章 + 鱼盆表格”转成可执行的趋势买入方向雷达。当前只做方向判断，不做单标的操作建议。

当前只专精一种买法：

```text
趋势买入 = 鱼盆前排 + 站上20日线 + 板块强于大盘
```

### 固定判定口径

| 条件 | 可复算定义 |
|---|---|
| 鱼盆前排 | 当前表格排名第1至第3 |
| 站上20日线 | 20日线偏离率严格大于0 |
| 板块/指数强于大盘 | 当日涨幅严格大于所属市场基准涨幅 |
| 连续性确认 | 最近两个鱼盆数据日都排名前三 |
| 量能确认 | 当前量比大于或等于1.00 |
| 短线偏热 | 20日线偏离率大于或等于8%，方向可以成立，但动作必须标记“不追高” |

基准固定为：

- A股指数和A股板块：中证A500。
- 港股指数：恒生指数。
- 美股指数：标普500。
- 日韩、黄金、白银等暂时只做环境观察，不进入趋势方向池。

文章判断是独立确认层：三条件、连续性和量能都通过，但最新文章没有支持该方向时，只能进入“试探”，不能进入“主攻”。

## 1. 新文章更新流

输入：你在对话中提供的新增公众号链接，或已追加到 `data/maobidao-link-log.md` 的链接。

流程：

```text
读取新增链接
→ 更新 data/maobidao-link-log.md
→ 抓取正文到 data/maobidao_articles.json
→ 若文章包含鱼盆表，抓取图片到 data/yupen-images/
→ 分别核对指数表/板块表数据日并更新 data/yupen-observations.csv
→ 更新 data/yupen-index-ranking.csv
→ 更新 docs/action-board.md
→ 追加 docs/reading-report.md
→ 若方法论变化，更新 docs/yupen-guide.md
```

产出：

- 当前方向雷达写入 `docs/action-board.md`
- 历史分析写入 `docs/reading-report.md`
- 方法规则写入 `docs/yupen-guide.md`

## 2. 趋势方向雷达

输入：最新猫笔叨文章判断；若文章包含鱼盆表，再加入鱼盆指数表和板块表。没有鱼盆表是合法状态，只归档文章，不改动当前方向看板。

必须先区分三个日期字段：

```text
文章发布日期：
指数鱼盆数据日：
板块鱼盆数据日：
```

两张表的数据日分别记录，不能默认相同。方向池以对应表格的数据日为准，文章用于解释和确认趋势含义。

先提取文章判断：

```text
市场状态：
作者看好的方向：
作者担心的风险：
需要验证的变化：
```

再读取鱼盆表格：

```text
指数/板块排名：
是否站上20日线：
偏离率：
排名变化：
是否强于大盘：
```

输出方向池：

| 分组 | 含义 |
|---|---|
| 主攻 | 三条件成立，连续两期前三，量比不低于1，且最新文章支持 |
| 试探 | 三条件成立，但连续性、量能或文章支持至少一项待确认 |
| 持有不追 | 排名前三、站上20日线且连续性仍在，但当日未强于基准；趋势未破，不新增 |
| 等待 | 只有部分条件改善，尚未形成趋势买入 |
| 回避 | 排名跌出前三且位于20日线下 |

方向分组与买点动作分开判断。例如一个方向可以是“主攻方向”，同时因为偏离率过高而标记为“不追高”。

## 4. 当前产品边界

不做：

- 泛新闻分析
- 实时行情
- 自动交易
- 主力资金猜测
- 回踩买入、突破买入、事件催化买入
- 单个股票操作建议

恢复 Web 前，先让“趋势方向雷达”连续跑通几次。
